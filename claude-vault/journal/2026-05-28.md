## Решения
- Без Instagram API (автопостинг требует Meta Business + модерацию — избыточно для старта). Постинг вручную.
- Данные в localStorage — приватно, на устройстве пользователя.

## На чём остановились
- Панель отдана пользователю (SendUserFile), запушено в `claude/instagram-blog-management-bSwru`, создан черновик PR #4.
- Проблема: пользователь открыл файл внутри приложения/мессенджера — там не работает JS, кнопки мёртвые. Решение: открыть в настоящем браузере (Safari/Chrome) или выложить по ссылке.
- Выбран способ деплоя по ссылке: **отдельный публичный репозиторий** (память в Gh остаётся приватной). Дал пошаговую инструкцию (создать публичный репо `vikablog` → залить index.html → включить Pages → ссылка `skyworkerq-hash.github.io/vikablog/`). Делает пользователь сам (мой MCP ограничен только репо Gh, чужой создать/пушить не могу).
- Просмотр контента IG с сервера невозможен: agent-browser не ставится (сетевой прокси с подменой TLS-сертификата не даёт скачать Chromium, готового браузера в контейнере нет), сам IG и зеркала (picuki/imginn) режут серверные запросы (403). Решение для пользователя — вьюеры с телефона.
- Добавил в панель вкладку **«Упаковка профиля»**: позиционирование, имя для поиска, шапка (RU+EN), мультиссылка, хайлайтс, закреплённые посты, визуал, аватар, tone of voice, чек-лист. Всё с черновиками и автосохранением в localStorage.
