## Решения

- **PDF-редактор живёт в отдельном публичном репо** `pdf-editor`, а не в основном `gh`. Причины: основной репо приватный (там память + личные данные клиентов), Pages с приватного требует Pro; для PDF-редактора нужен публичный URL, чтобы пользователь мог пользоваться с любого устройства без скачивания.
- **API-ключ DeepSeek — только в localStorage браузера.** Не в коде, не в репо, не в URL. Каждый пользователь приложения сам вставляет ключ.
- **Извлечение реквизитов из текстового слоя, не OCR.** Большинство договоров приходят как PDF с текстовым слоем — это быстрее и точнее. OCR — fallback с явным подтверждением (медленно: 10-30 сек/страница).
- **Промпт-схема — фиксированный JSON.** Поля заточены под бизнес агента по БГ: ИНН/КПП/ОГРН сторон, реквизиты БГ, ссылка на ФЗ. Если поле не найдено — `null`, не выдумываем.
- **`deepseek-chat` по умолчанию.** Быстрее и дешевле; `reasoner` — опция для сложных договоров.
- **PR #1 пока draft.** Переведём в Ready после прогона на реальном договоре и фидбэка.
