# Claude Vault — постоянная память

Это персональный «второй мозг» для Claude Code между сессиями. Контейнер в облаке эфемерный, поэтому всё, что нужно помнить, хранится здесь в git.

## Инструкции для Claude (читай в начале каждой сессии)

1. **Прочитай `knowledge/context.md`** — общий контекст о пользователе и работе.
2. **Прочитай `knowledge/preferences.md`** — предпочтения по стилю кода, инструментам, языку общения.
3. **Просмотри `projects/`** — список активных проектов. Открой `README.md` нужного проекта перед началом работы над ним.
4. **Загляни в последнюю запись `journal/`** — что делали в прошлую сессию, на чём остановились.

## Структура

```
claude-vault/
├── CLAUDE.md              ← этот файл, точка входа
├── projects/              ← по папке на каждый проект
│   └── <project-name>/
│       ├── README.md      ← описание, цели, контекст
│       ├── notes.md       ← заметки, идеи, ход мыслей
│       └── decisions.md   ← принятые решения и обоснования
├── knowledge/
│   ├── preferences.md     ← предпочтения пользователя
│   ├── context.md         ← общий контекст
│   └── snippets.md        ← полезные команды и куски кода
└── journal/
    └── YYYY-MM-DD.md      ← лог сессий по датам
```

## Правила ведения памяти

- **В конце сессии** обновляй: журнал за сегодня, заметки/решения по затронутым проектам, при необходимости — knowledge.
- **Решения пиши с обоснованием** («почему», а не только «что») — чтобы не передумывать заново.
- **Журнал — кратко**: что сделали, на чём остановились, что дальше.
- **Коммить и пушь** изменения памяти перед завершением сессии, иначе всё пропадёт.

## Команды-триггеры от пользователя

- «обнови память» / «сохрани контекст» → записать актуальное состояние во все нужные файлы и закоммитить.
- «загрузи проект X» → прочитать `projects/X/` целиком.
- «что было вчера/в прошлый раз» → последняя запись в `journal/`.
