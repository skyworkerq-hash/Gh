# Wearable Assistant — носимый AI-кулон на OpenClaw

> Рабочее название. Имя ассистента пользователь придумает позже.

## Идея

Носимое устройство (кулон/клипса), которое:
- Всегда с тобой, слышит и говорит.
- Имеет память (Obsidian-vault + векторная БД).
- Через AI-агента (DeepSeek V4) выполняет действия: бронь столов,
  заказ еды, такси, погода, заметки, помощь по работе с банковскими
  гарантиями.

«Свой Jarvis» под бизнес агента по БГ + бытовые дела.

## Главное решение: строим на OpenClaw

Не пишем агентский фреймворк с нуля. Берём **OpenClaw**
(<https://github.com/openclaw/openclaw>) — open-source личный AI-ассистент
Питера Штайнбергера, самый звёздный проект на GitHub в 2026 году.

**Что получаем «из коробки»:**
- 24 канала общения (Telegram, WhatsApp, iMessage, Signal, …).
- «Eyes and hands»: ходит в браузер, читает/пишет файлы, гоняет shell.
- Voice Wake + push-to-talk на macOS/iOS/Android.
- TTS через ElevenLabs.
- Скилы в `~/.openclaw/workspace/skills/` — туда добавляем своё.
- DeepSeek подключается как кастомный OpenAI-совместимый провайдер.
- MIT-лицензия — модифицируем как хотим.

## Главное решение: на ESP32 ставим MimiClaw

Для финального кулона — **MimiClaw** (<https://github.com/memovai/mimiclaw>):
прошивка для ESP32-S3, которая повторяет API OpenClaw на чипе за $5.
Без Linux, без Node.js. Создана специально для wearable.

Альтернативы на запас: ESP-Claw (от Espressif, апрель 2026) и WireClaw.

## Известный блок: русский STT

OpenClaw Talk Mode пока умеет только en-US (Issue #44688).
**Обход:** не используем встроенный STT. Свой pipeline на Whisper
(понимает русский отлично) → текст подаём в OpenClaw как обычное
сообщение. +2-3 дня работы, не блокер.

## Архитектура

```
[Кулон на ESP32-S3, прошивка MimiClaw]
  ├── Микрофон → захват голоса
  ├── Wi-Fi/BLE → передача аудио
  └── Динамик → проигрывание ответа

                |
                v

[Mac или VPS, OpenClaw + наши надстройки]
  ├── Кастомный STT-pipeline:
  │   └── Whisper (faster-whisper локально или OpenAI API) — русский
  ├── OpenClaw (Node.js):
  │   ├── DeepSeek V4 как кастомный провайдер
  │   ├── Скилы:
  │   │   ├── обвязки Yandex Taxi / Yandex Maps / Yandex Eda
  │   │   ├── бронь столов через browser-agent
  │   │   ├── работа с tender-monitor
  │   │   ├── разбор документов (через наш PDF-редактор)
  │   │   └── append_to_obsidian, reminders, etc.
  │   └── Telegram-канал (для теста и как запасной интерфейс)
  ├── Память:
  │   ├── Obsidian-vault (Markdown) — синхронизация через Yandex Disk
  │   └── Chroma (векторная БД) для семантического поиска
  └── TTS:
      ├── Yandex SpeechKit (повседневка, дёшево, нативный русский)
      └── ElevenLabs (опционально, для клонирования голоса)
```

## План по стадиям (обновлённый)

### Стадия 0 — Подготовка (сейчас)
- [x] Завести проект.
- [x] Выбрать стек: OpenClaw + MimiClaw + Whisper-русский-обход.
- [ ] Получить рабочий API-ключ DeepSeek.
- [ ] Получить API-ключ Yandex SpeechKit.
- [ ] Пользователь придумывает имя ассистенту.

### Стадия 1 — MVP без железа (1 неделя, можно начать сразу)
**Цель:** работающий ассистент в Telegram, голос+текст, на твоём Mac или VPS.

- [ ] Установить OpenClaw на Mac (Node 22+).
- [ ] Подключить DeepSeek V4 как кастомный провайдер.
- [ ] Подключить Telegram-канал OpenClaw.
- [ ] Поставить наш Whisper-pipeline (faster-whisper локально, бесплатно).
- [ ] Привязать Yandex SpeechKit для TTS.
- [ ] Написать первые 3 скила: `append_to_obsidian`, `set_reminder`, `weather`.
- [ ] Проверить русский голосом в Telegram → DeepSeek → ответ голосом.

**Что НЕ делаем:** железо, заказ еды, бронь столов, wake-word.

### Стадия 2 — Кулон (после стадии 1, +3-4 недели)
**Цель:** перевести то же самое в компактный кулон.

- [ ] Заказать ESP32-S3-BOX-3 (отладочная плата) — отлаживаем без пайки.
- [ ] Прошить MimiClaw на BOX-3, проверить связку с нашим сервером.
- [ ] Заказать компоненты под кулон (см. shopping-list-stage-2.md, позже).
- [ ] Спаять.
- [ ] Спроектировать и напечатать корпус.

### Стадия 3 — Действия в мире (после стадии 2)
- [ ] Скил `book_taxi` — Yandex Taxi API (есть официальный).
- [ ] Скил `find_route` — Yandex Maps API.
- [ ] Скил `order_food` — через browser-agent (нет публичного API Yandex Eda).
- [ ] Скил `book_restaurant` — через browser-agent.
- [ ] Wake-word через OpenWakeWord / Porcupine.

### Стадия 4 — Помощник по работе (параллельно)
- [ ] Скил `lookup_company_by_inn` — rusprofile.
- [ ] Скил `tender_monitor` — связка с другим нашим проектом.
- [ ] Скил `parse_document` — разбор PDF через DeepSeek.
- [ ] Скил `draft_bank_request` — шаблоны писем в банки + humanizer.

## Кто что делает

- **Пользователь:** заказы железа, пайка по инструкциям, тесты,
  формулирование задач для ассистента.
- **Claude:** установка OpenClaw, написание скилов на TypeScript/Python,
  Whisper-pipeline, ESP32-прошивка через MimiClaw, инструкции по
  пайке, чертежи корпуса.

## Открытые вопросы

1. **Имя ассистента.**
2. **Сервер на Mac или VPS** — на стадии 1 проще Mac, на стадии 2 нужен 24/7.
3. **TTS-голос** — Yandex базовый, или клонировать через ElevenLabs.
