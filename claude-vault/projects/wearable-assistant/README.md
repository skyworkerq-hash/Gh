# Wearable Assistant — носимый AI-кулон

> Рабочее название проекта. Финальное имя ассистента пользователь придумает позже.

## Идея

Носимое устройство (кулон/клипса на одежде), которое:
- Всегда с тобой, постоянно «слышит».
- Распознаёт голос, отвечает голосом.
- Имеет память (Obsidian-vault + векторная БД).
- Через AI-агента (DeepSeek V4 API) умеет совершать действия:
  бронировать столы, заказывать еду, вызывать такси, проверять
  погоду, записывать заметки, ставить напоминания, помогать с
  работой по банковским гарантиям.

«Свой Jarvis из Iron Man», только под бизнес агента по БГ +
бытовые дела.

## Аналоги, на которые ориентируемся

- **OMI (бывш. Friend Dev Kit)** — open-source носимый AI на ESP32.
  Схемы и прошивка открыты, можно собрать самим. <https://github.com/BasedHardware/omi>
- **xiaozhi-esp32** — китайский open-source AI-агент для ESP32,
  поддерживает DeepSeek/OpenAI из коробки.
- **Limitless Pendant**, **Bee AI**, **Plaud NotePin** — коммерческие
  закрытые варианты, не подходят (мы хотим открытость и кастомизацию).

## Архитектура (планируемая)

```
[Кулон на ESP32-S3]
  ├── Микрофон I2S → захват голоса
  ├── Wi-Fi/BLE → отправка аудио и команд
  ├── Динамик/наушник → проигрывание ответа
  └── Кнопка / wake-word → активация

           |
           v (по Wi-Fi или через телефон-мост)

[Python-сервер (VPS или Mac)]
  ├── STT: Whisper (OpenAI API или faster-whisper локально)
  ├── LLM: DeepSeek V4 Pro / Flash через API
  ├── TTS: Yandex SpeechKit (русский, дёшево) или ElevenLabs
  ├── Память:
  │   ├── Obsidian-vault (Markdown-файлы, синхронизация через Yandex Disk)
  │   └── Векторная БД (Chroma или Qdrant) для семантического поиска
  └── Агент-фреймворк (на выбор: Claude Agent SDK / OpenAI Agents SDK / свой)
      └── Инструменты:
          ├── append_to_obsidian(text)
          ├── set_reminder(when, what)
          ├── weather(city)
          ├── book_taxi(from, to)        ← Yandex Taxi API
          ├── find_route(from, to)       ← Yandex Maps API
          ├── translate(text, target_lang)
          ├── search_web(query)
          ├── order_food(...)            ← browser-agent
          ├── book_restaurant(...)       ← browser-agent
          └── work_helpers (БГ-специфичные, постепенно)
```

## План по стадиям

### Стадия 0 — Подготовка (сейчас)
- [x] Завести проект, зафиксировать решения.
- [ ] Дождаться выбора пользователя: имя ассистента, отправить заказ железа.
- [ ] Получить рабочий API-ключ DeepSeek (текущий скомпрометирован).

### Стадия 1 — Прототип на готовой плате (1–2 недели)
**Цель:** заставить плату слышать → передавать в DeepSeek → отвечать голосом.

- [ ] Заказать **ESP32-S3-BOX-3** (см. `shopping-list-stage-1.md`).
- [ ] Установить инструменты разработки: ESP-IDF или Arduino IDE.
- [ ] Прошить базовую прошивку: запись → отправка по Wi-Fi.
- [ ] Поднять Python-сервер на VPS / Mac:
  - HTTP/WebSocket endpoint для приёма аудио.
  - Whisper → DeepSeek → Yandex SpeechKit.
  - Отправка аудио-ответа обратно на плату.
- [ ] Простейшая память: записывать диалоги в Markdown-файлы.

**Что НЕ делаем на этой стадии:**
- ❌ Пайка (всё в готовой плате).
- ❌ Wake-word (пока по кнопке).
- ❌ Заказ еды / бронь столов (это стадия 3).
- ❌ Корпус кулона.

### Стадия 2 — Кулон (2–3 недели после стадии 1)
**Цель:** перевести то же самое в компактный носимый форм-фактор.

- [ ] Заказать компоненты для кулона (см. отдельный shopping-list, позже).
- [ ] Спаять плату (или поручить).
- [ ] Спроектировать корпус (3D-печать).
- [ ] Перенести прошивку.
- [ ] Батарея + зарядка через USB-C.

### Стадия 3 — Действия в мире (после стадии 2)
**Цель:** реально заказывать и бронировать.

- [ ] Интеграция Yandex Taxi API (есть официальный).
- [ ] Интеграция Yandex Maps API (есть официальный).
- [ ] browser-agent для Yandex Eda / Delivery Club (нет API → через браузер).
- [ ] browser-agent для бронирования столов (Tomesto / Restoplace).
- [ ] Wake-word через OpenWakeWord (распознавание «Привет, <имя>!»).

### Стадия 4 — Помощник по работе (параллельно)
- [ ] Голосовой ввод реквизитов клиента → ИНН/КПП → данные с rusprofile.
- [ ] Связка с tender-monitor: «Симон, что нового по тендерам?»
- [ ] Разбор PDF договоров (через наш PDF-редактор + DeepSeek).

## Кто что делает

- **Пользователь:** заказывает железо, паяет (после инструкций),
  тестирует, формулирует, что должен уметь ассистент.
- **Claude:** проектирует, пишет инструкции по сборке, пишет прошивку
  и серверный код, готовит чертежи корпуса под печать, отлаживает.

## Открытые вопросы

1. **Имя ассистента** — пользователь придумывает.
2. **Где будет крутиться сервер** — Mac, VPS (Hetzner/Yandex Cloud),
   локально дома? От этого зависит wi-fi-схема и латентность.
3. **TTS-голос** — Yandex SpeechKit (дёшево, нейтральный русский)
   или ElevenLabs (дороже, но можно клонировать любой голос)?
4. **Wake-word или кнопка** — кнопка надёжнее, wake-word удобнее.
   На прототипе — кнопка, на финальном кулоне обсудим.
