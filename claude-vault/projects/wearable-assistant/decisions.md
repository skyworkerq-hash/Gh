# Решения по проекту Wearable Assistant

## 2026-05-16

### Форм-фактор: носимое устройство (кулон)
**Решение:** не делаем мобильное приложение, не делаем колонку,
не делаем Telegram-бота как финальный продукт. Цель — кулон.
**Почему:** пользователь хочет реально носимое устройство, чтобы
ассистент был «всегда с тобой». Это отличает проект от обычных
чатботов.

### Собираем сами, не покупаем готовое
**Решение:** не покупаем Friend, Bee, OMI готовыми. Собираем DIY.
**Почему:**
- Полный контроль над прошивкой и протоколом.
- Можем выбрать любой LLM (DeepSeek, а не привязка к их сервису).
- Можем хранить данные у себя в vault (приватность).
- Дешевле (~2 500₽ итого против $70–$169 готовых).
- Образовательная ценность — пользователь учится паять.

### Двухстадийная сборка: dev-плата → кулон
**Решение:** сначала отлаживаем всю программную часть на готовой
плате ESP32-S3-BOX-3, потом переносим в компактный кулон.
**Почему:**
- Пользователь — новичок в пайке. На стадии 1 паять не нужно вообще.
- Сначала отлаживаем мозг (где 80% работы), потом упаковка.
- Если на стадии 1 что-то не работает — это софт, а не сломанная
  пайка. Меньше переменных для отладки.

### Микроконтроллер: ESP32-S3
**Решение:** не Raspberry Pi, не Arduino, не nRF52.
**Почему:**
- Есть Wi-Fi и Bluetooth из коробки.
- AI-ускоритель для on-device wake-word.
- Дешёвый, ~600₽ голый чип.
- Огромная open-source экосистема (xiaozhi, WillowVoice, ESPHome).
- Низкое энергопотребление в deep sleep — годится для кулона на батарейке.

### LLM: DeepSeek V4
**Решение:** ставим на DeepSeek V4 Pro/Flash.
**Почему:**
- 1M контекст — можно засовывать всю память Obsidian при необходимости.
- Дёшево (~10× дешевле GPT-4).
- Тестовый ключ уже есть (но скомпрометирован, нужен новый).
- Возможна замена на GPT-4 или Claude через тот же API-интерфейс.

### TTS: пока Yandex SpeechKit
**Решение:** русский голос через Yandex SpeechKit.
**Почему:**
- Дёшево.
- Качество приемлемое для русского (ElevenLabs русский всё ещё слабее).
- Если захотим клонировать голос — переключимся на ElevenLabs.

### STT: Whisper
**Решение:** OpenAI Whisper (API или локально через faster-whisper).
**Почему:**
- Один из лучших по русскому.
- Open-source локальная версия — приватность.
- Если на сервере мощно — крутим локально и не платим.

### Где работает сервер
**Решение:** на стадии 1 — на Mac пользователя или на бесплатном VPS
для тестов. Решим конкретно по ходу.
**Почему:** для одного пользователя ресурсов нужно немного.

### Память: гибрид Obsidian + векторная БД
**Решение:** все заметки и диалоги в Markdown-файлах в Obsidian-vault,
поверх — векторная база (Chroma локально) для семантического поиска.
**Почему:**
- Markdown переживёт любую смену стека.
- Obsidian — то, чем пользователь уже владеет на Mac.
- Векторная БД для поиска «о чём мы говорили месяц назад».

### Имя ассистента — отложено
**Решение:** пользователь придумает имя сам, отдельно.
До этого — рабочее название `wearable-assistant`.
