# Решения по проекту Office Assistant (бывш. Wearable Assistant)

## 2026-05-16 — стартовые решения (исторические)

### Форм-фактор: носимое устройство (кулон) ❌ ОТМЕНЕНО 2026-05-17
**Решение:** не делаем мобильное приложение, не делаем колонку,
не делаем Telegram-бота как финальный продукт. Цель — кулон.
**Почему:** пользователь хочет реально носимое устройство.
**❌ Пересмотрено 2026-05-17:** см. ниже.

### Собираем сами, не покупаем готовое ✅ В СИЛЕ
**Решение:** не покупаем Friend, Bee, OMI готовыми. Собираем DIY.
**Почему:** контроль, приватность, выбор LLM, образовательная ценность.

### Микроконтроллер: ESP32-S3 ❌ ПЕРЕСМОТРЕНО 2026-05-17
**Решение:** ESP32-S3.
**❌ Пересмотрено:** перешли на ESP32-P4 — следующее поколение,
RISC-V, Wi-Fi 6, MIPI-CSI для камеры. См. ниже.

### LLM: DeepSeek V4 ✅ В СИЛЕ
DeepSeek V4 Pro/Flash, 1M контекст, дёшево, через кастомный
OpenAI-совместимый API.

### TTS: Yandex SpeechKit ✅ В СИЛЕ
Русский голос, дёшево, нейтральный.

### STT: Whisper ✅ В СИЛЕ
OpenAI Whisper или локально faster-whisper. Лучший по русскому.

### Память: гибрид Obsidian + векторная БД (Chroma) ✅ В СИЛЕ
Markdown переживёт смену стека, векторная БД для поиска по
прошлым диалогам.

## 2026-05-16 (поздний вечер) — OpenClaw как фреймворк

### Агент-фреймворк: OpenClaw ✅ В СИЛЕ
**Решение:** не пишем агент-фреймворк сами. Берём OpenClaw
(<https://github.com/openclaw/openclaw>).
**Почему:**
- 250k звёзд на GitHub (самый звёздный проект 2026), активная разработка.
- MIT-лицензия.
- 24 канала общения готовы (Telegram/iMessage/WhatsApp/...).
- «Eyes and hands» из коробки: браузер, файлы, shell.
- Skills экосистема в `~/.openclaw/workspace/skills/`.
- DeepSeek подключается как кастомный OpenAI-совместимый провайдер.

### Прошивка ESP32: MimiClaw ❌ НЕ ПОДХОДИТ ДЛЯ P4 (2026-05-17)
**Решение 2026-05-16:** взять MimiClaw, готовую прошивку под
ESP32-S3, повторяющую API OpenClaw на чипе.
**❌ Пересмотрено 2026-05-17:** перешли на ESP32-P4, MimiClaw
под P4 пока не поддерживается. См. ниже.

### Русский STT в OpenClaw: обход через свой Whisper-pipeline ✅ В СИЛЕ
**Проблема:** Talk Mode OpenClaw поддерживает только en-US
(Issue #44688).
**Решение:** не используем встроенный STT. Захват аудио →
наш Whisper-pipeline → текст подаём в OpenClaw как сообщение.

## 2026-05-17 — пересмотр форм-фактора и железа

### Форм-фактор: стационарный AI-хаб в офисе ✅ ФИНАЛЬНО
**Решение:** не кулон, не часы, не Telegram, не колонка. **Хаб с
круглым экраном на рабочем столе** в офисе.
**Почему:**
- Пользователь — агент по БГ, работает в офисе. Большую часть
  времени на одном рабочем месте.
- Стационарка снимает все wearable-проблемы: батарея, размер,
  моно-микрофон, шнурок, корпус.
- Можно ставить мощный чип (P4), большой экран, камеру.
- Камера + микрофоны far-field открывают сценарии «положи
  документ перед устройством — расскажи что это», «узнай
  клиента который подошёл».

### Железо: ESP32-P4-WIFI6-Touch-LCD-3.4C/4C ✅ ФИНАЛЬНО
**Решение:** Waveshare ESP32-P4-WIFI6-Touch-LCD-3.4C (3.4″ 800×800)
или 4C (4″ 720×720). Артикул AliExpress **`1005009167855279`**.
**Почему:**
- Новейший RISC-V SoC от Espressif (2025).
- Wi-Fi 6 — устаревать не будет.
- 2 микрофона + ES7210 echo cancellation — far-field.
- MIPI-CSI готов под камеру.
- 40-pin GPIO как Pi — навешиваем GPS, датчики.
- Большой круглый IPS-экран с тачем — красиво в офисе.
- Производитель явно заявил поддержку DeepSeek.
- ~7 500 ₽ за плату.

**Минусы и почему мы их принимаем:**
- MimiClaw пока не работает на P4 → пишем прошивку сами через
  ESP-IDF + ESP-SR. +1-2 месяца разработки.
- Не носимое → но мы и не хотим носимое.
- Питание от USB-C → в офисе это не проблема.

### Камера ✅ ПЛАН
**Решение:** Waveshare SC2336 (родная для P4) или OV5640
(универсальная). Брать сразу при заказе платы.
**Зачем:** OCR договоров, распознавание клиентов, визуальный AI.

### GPS — опционально ✅ ПЛАН
**Решение:** GPS-модуль NEO-M8N (~$10 на Ali) — **берём, но как
nice-to-have**.
**Когда нужен:** если устройство берётся на встречи / стоит у окна.
**Когда не нужен:** в центре офиса GPS не ловит — лучше вшить
адрес офиса в конфиг ассистента.

### Прошивка под P4: ESP-IDF + ESP-SR ✅ ПЛАН
**Решение:** пишем прошивку с нуля на ESP-IDF (официальный фреймворк
Espressif), используя ESP-SR (Espressif Speech Recognition stack)
для wake-word и аудио-pipeline. Не используем MimiClaw.
**Почему:**
- MimiClaw под P4 не готов, неизвестные сроки.
- ESP-SR официальный, поддерживается, отлажен.
- Под P4 уже есть референсы Espressif для voice AI.
- Слой связи с нашим OpenClaw-сервером пишем сами поверх
  WebSocket — это ~2-3 недели работы.
- Минус — больше нашего кода, чем если бы был MimiClaw.

### Имя ассистента — отложено ✅ В СИЛЕ
Пользователь придумает имя сам, отдельно.

### Сервер на Mac или VPS — пока Mac, потом VPS
**Решение:** стадия 1 (тесты в Telegram) — на Mac пользователя.
Когда P4 будет стоять в офисе 24/7 — переезжаем на VPS, чтобы
ассистент не зависел от Mac.
**Когда переедем:** при первом случае «закрыл Mac → ассистент
умер». Тогда возьмём Yandex Cloud (карта РФ) или Hetzner.

## Двухстадийный путь к рабочему продукту

**Стадия 1 (1-2 недели):** OpenClaw + DeepSeek + Telegram-канал.
Уже можно реально пользоваться через мессенджер. Без железа.

**Стадия 2 (после получения P4, +1-2 месяца):** прошивка P4
+ перенос всего на железо в офис.

Это даёт **первые рабочие результаты через 2 недели**, а не
через 3 месяца молчания.
