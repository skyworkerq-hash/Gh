# PDF-редактор + AI-извлечение реквизитов

Single-file веб-приложение «как UPDF» для работы с PDF-договорами агента по банковским гарантиям. Работает целиком в браузере, без бэкенда.

## Где живёт

- **Рабочая ссылка:** <https://skyworkerq-hash.github.io/pdf-editor/>
- **Публичный репо приложения:** <https://github.com/skyworkerq-hash/pdf-editor> (только `index.html`)
- **Источник правды в разработке:** `apps/pdf-editor/index.html` в основном приватном репо `skyworkerq-hash/gh`. Оттуда вручную копируется в публичный репо для деплоя.

## Что умеет

1. **Просмотр PDF** — навигация, зум, миниатюры страниц.
2. **Аннотации** — подсветка, рисование, текст. Сохраняются при экспорте в PDF.
3. **OCR** — распознавание текста на странице (русский + английский, Tesseract.js, ~15 МБ модели).
4. **AI-извлечение реквизитов** — через DeepSeek API:
   - Стороны договора: наименование, ИНН, КПП, ОГРН, адрес.
   - Номер, дата, предмет, сумма договора.
   - ФЗ (44/223), номер закупки/ИКЗ.
   - Срок, место исполнения.
   - Требования к банковской гарантии: тип, %, сумма, срок.
5. **Экспорт реквизитов** — TSV для вставки в Excel или JSON.
6. **Скачивание PDF** с аннотациями.

## Технологии

- `pdf.js` (просмотр и извлечение текстового слоя)
- `pdf-lib` (экспорт с аннотациями)
- `Tesseract.js` (OCR rus+eng)
- DeepSeek API (`deepseek-chat` / `deepseek-reasoner`)
- Чистый JS + ESM-импорты с jsdelivr CDN
- 0 бэкенда

## Безопасность

- API-ключ DeepSeek — **только в localStorage браузера пользователя** (`pdfeditor.deepseek.apiKey`).
- В коде/репо/URL ключа нет.
- PDF-файлы клиента не покидают браузер до момента отправки текста в DeepSeek API.
- DeepSeek получает только **текст** договора (не сам PDF), и только когда пользователь жмёт «Извлечь реквизиты».

## Под кого заточено

Бизнес-кейс пользователя: агент по банковским гарантиям. Каждый клиент приносит договор → нужно вытащить реквизиты для заявки в банк. Раньше это было руками, теперь — одной кнопкой.

## Что протестировать в первую очередь

1. **Реальный договор клиента** — открыть, нажать AI-реквизиты, сверить с тем, что в самом договоре.
2. **Договор без текстового слоя** (отсканированный) — должен предложить OCR.
3. **Большой договор (50+ стр.)** — проверить, что 60К-лимит текста не режет важное.
4. **Договор 223-ФЗ vs 44-ФЗ** — разные шапки, должна правильно классифицироваться.
