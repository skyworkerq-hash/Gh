# Решения по PDF-редактору

## 2026-05-15
- **Single-file HTML без бэкенда.** Чтобы пользователь мог просто открыть файл локально, без установки чего-либо. Минус — клиентский OCR (~15 МБ модели грузится в браузер).
- **ESM-импорты с jsdelivr CDN.** Не таскаем библиотеки в репо, экономим место. Минус — без интернета не запустится.
- **Аннотации только в трёх вариантах** (подсветка, рисование, текст). Хватает для рабочего использования; больше — усложнение без явной пользы сейчас.

## 2026-05-16
- **AI-ключ — в localStorage, не в коде.** Это публичное веб-приложение, ключ нельзя хардкодить.
- **Реквизиты только через текстовый слой, OCR — опционально.** Текстовый слой — мгновенно и точно. OCR медленный (10-30 сек/страница) и менее точный; запускается только когда текста реально нет, и только с подтверждения пользователя.
- **Промпт со строгой JSON-схемой** под бизнес агента по БГ. Не «общий парсер договоров», а заточенный под конкретные поля, которые нужны для заявки в банк.
- **`temperature: 0.1` для извлечения.** Снижаем «креативность» — нам нужны факты, а не интерпретация.
- **`response_format: { type: "json_object" }`.** DeepSeek поддерживает структурированный вывод, используем для надёжности.
- **Лимит текста 60К символов.** Если договор больше — обрезаем середину, оставляем начало и конец (там обычно реквизиты сторон и подписи).
- **Отдельный публичный репо `pdf-editor` для деплоя.** Основной репо `gh` приватный — Pages в нём требует Pro. Создавать публичный sub-репо проще, чем платить.
- **Файл копируется руками между репо.** Пока правки нечастые, автоматизация не нужна. Если станет неудобно — добавим второй git remote и push-скрипт.
