# Tender Monitor — мониторинг тендеров

## Описание

Ежедневный мониторинг тендеров (госзакупки + коммерческие площадки) для поиска лидов — поставщиков, которым понадобится банковская гарантия. Утром пользователь получает отчёт с подходящими тендерами и их выигравшими поставщиками.

## Зачем

Сейчас пользователь тратит много времени на ручной поиск лидов. Автоматизация должна:
- Сократить время на поиск с часов до минут.
- Не пропустить «жирные» лиды.
- Отфильтровать мусор (мелкие/неподходящие тендеры).

## Цели (что должно получиться)

1. **Скрипт/агент**, который раз в день забирает свежие тендеры с указанных площадок.
2. **Фильтрация** по критериям:
   - Сумма контракта / НМЦК (диапазон уточнить).
   - Тип банковской гарантии (исполнения / гарантийных обязательств / возврата аванса).
   - Качество поставщика (опыт, выручка, история побед) — для тендеров, где победитель уже определён.
3. **Отчёт в ваулт** — раз в день, в `journal/` или в `projects/tender-monitor/reports/YYYY-MM-DD.md`.

## Контекст

- **Основная площадка для старта:** `zakupki.gov.ru` (44-ФЗ).
- **Другие площадки на потом:** Сбер-А, РТС-тендер, РОСЭЛТОРГ, B2B-Center (223-ФЗ и коммерческие); агрегаторы — Seldon, Kontur.Zakupki, Tenderplan.
- **Объём бизнеса пользователя:** 10–50 заявок/мес — значит из мониторинга должно отфильтровываться не больше нескольких десятков релевантных лидов в день, чтобы оставалось обозримо.
