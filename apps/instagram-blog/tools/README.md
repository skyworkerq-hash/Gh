# Инструменты SMM (браузерная способность Claude)

Здесь живёт то, что позволяет **мне (Claude) самому открывать сайты** в облачной
сессии: настоящий браузер Chromium через Playwright + готовые скрипты.

## Зачем
Контейнер облачной сессии эфемерный — браузер не переживает рестарт. Этот
каталог хранит **скрипт восстановления** и рабочие скрипты, чтобы в любой новой
сессии я за минуту вернул себе способность работать с вебом.

## Как поднять (в начале сессии, если нужен браузер)
```bash
bash apps/instagram-blog/tools/setup.sh
```
Ставит Playwright и Chromium. Работает несмотря на TLS-прокси среды, потому что
загрузчик Playwright (Node) уважает `NODE_EXTRA_CA_CERTS`.

## Скрипты
- `igapi.js <username>` — тянет профиль Instagram через публичный web-API.
  Сейчас Instagram отдаёт **401 require_login** для незалогиненных — без сессии не работает.
- `viewer.js <username>` — пробует зеркала-вьюеры (imginn, picuki, gramhir, greatfon, anonyig).
  С серверного дата-центрового IP почти все блокируются Cloudflare (403/451).

## Что выяснено про Instagram (2026-05)
- Профиль и публичный API IG **полностью за логином** — анонимно контент не отдаётся.
- Зеркала-вьюеры **режут серверные IP** анти-ботом. С телефона пользователя они работают.
- Вывод: автономный анонимный скрейп IG с облака невозможен честным путём.
  Рабочие пути для автономного SMM:
  1. **Instagram Graph API** — официально: публикация, расписание, статистика.
     Нужен Business/Creator-аккаунт + токен Meta. Это и есть «настоящий» SMM-доступ.
  2. **Залогиненная сессия в браузере** (cookie/вход пользователя) — мощно для «увидеть всё»,
     но это серая зона по ToS и чувствительно по данным. Только по явной просьбе пользователя.

## Браузер полезен и без Instagram
Chromium открывает любые другие сайты: скриншоты, заполнение форм, парсинг,
тестирование веб-приложений (включая нашу панель `index.html`).
